# Zen-Gamer Платформын Системийн Архитектур
*Дипломын төсөлд зориулсан хураангуй баримт бичиг*

---

## 1. Системийн Ерөнхий Бүтэц (Overview)

Zen-Gamer нь дараах технологийн стек дээр суурилсан цахим спортын маркетплейс платформ юм:
*   **Фронтенд (Frontend)**: React.js + Vite, Tailwind CSS, Framer Motion (Урсах хөдөлгөөнүүд).
*   **Бэкенд (Backend)**: Express.js (Node.js) - Vercel Serverless орчинд ажиллана.
*   **Өгөгдлийн сан (Database)**: NeonDB (Serverless PostgreSQL).
*   **Хадгалалт (Storage)**: Cloudinary (Зураг хадгалах).
*   **AI систем**: Google Gemini API (Ухаалаг туслах).

---

## 2. Архитектурын Хэв Шинж (Monolith & Decoupled Architecture)

*   **Фронтенд ба Бэкендийн Салгалт (Decoupled SPA)**: Фронтенд болон Бэкенд нь хоорондоо бүрэн хамааралгүй тусдаа ажиллаж, зөвхөн REST API-аар холбогдоно.
*   **Монолит бэкенд (Monolithic API)**: Бэкенд логикууд (Хэрэглэгчид, Бараа, Захиалга, Дундын данс, Чат) нь тусдаа микросервист хуваагдаагүй, ганц кодны санд (`api/index.mjs`) багтсан **Монолит** бүтэцтэй. Энэ нь өгөгдлийн нэгдмэл байдлыг хангаж, системийн ажиллагааг хялбар, найдвартай болгодог.
*   **Сервергүй ба Төлөвгүй (Serverless & Stateless)**: Сервер нь Vercel-ийн Serverless функц хэлбэрээр ажиллах тул төлөвгүй (Stateless). Бүх сешн, өгөгдлүүд зөвхөн NeonDB өгөгдлийн санд болон JWT токенд хадгалагдана.

---

## 3. Өгөгдлийн Сангийн Бүтэц (Database Schema)

*   **`users` (Хэрэглэгчид)**: Хэрэглэгчийн нэр, и-мэйл, шифрлэсэн нууц үг (bcrypt), баланс, профайл мэдээлэл болон эрх (`OPERATOR`, `moderator`, `admin`, `owner`).
*   **`products` (Бараа/Үйлчилгээ)**: Зарагдаж буй үйлчилгээ болон техник хангамжийн мэдээлэл, 3 төрлийн багцын үнэ (`basic`, `pro`, `elite`), зурагны холбоосууд.
*   **`orders` (Захиалга)**: Захиалгын ерөнхий бүртгэл, сонгосон багц, гүйлгээний дүн болон явцын төлөв.
*   **`escrow_trades` (Дундын данс)**: Худалдагчийн аккаунтын мэдээлэл, аюулгүй байдлыг хангах мөнгө түгжих хэсэг, шалгаж буй модераторын тэмдэглэл.
*   **`conversations` & `messages` (Чат)**: Хэрэглэгч болон борлуулагч хоорондын харилцааны түүх.
*   **`notifications` (Мэдэгдэл)**: Системээс хэрэглэгчид очих мэдэгдлүүд.

---

## 4. Баталгаажуулалт болон Аюулгүй Байдал (Security & Auth)

*   **JWT (JSON Web Token)**: Нэвтрэхэд хэрэглэгчийн ID болон эрхийг агуулсан токен үүсэж, хөтөчийн `localStorage`-д хадгалагдан API хүсэлт болгонд Header-ээр дамжина.
*   **Нууц үг шифрлэлт**: Хэрэглэгчийн нууц үгийг баазад хадгалахын өмнө `bcrypt` ашиглан нэг чиглэлт хэш (hash) болгон хувиргана.
*   **Role-Based Authorization**: Админ/модераторууд хэрэглэгчийг хориглох эсвэл гүйлгээг баталгаажуулах зэрэг тусгай эрхүүдтэй.

---

## 5. Фронтенд болон Бэкендийн Холболт (Fetch API Wrapper)

Програмд гадны том сангуудаас (жишээ нь `axios`) хамааралтай байдлыг багасгаж, системийн ачаалах хурдыг нэмэгдүүлэх зорилгоор хөтөчийн стандарт **`Fetch API`** ашигласан.

1.  **Төвлөрсөн `request` модуль (`src/lib/api.ts`)**:
    *   **Автомат JWT**: Хүсэлт бүрийн `Authorization: Bearer <token>` толгойг автоматаар хавсаргана.
    *   **Content-Type удирдлага**: Энгийн өгөгдлийг JSON хөрвүүлж, зургийн файлыг `FormData` хэлбэрээр илгээнэ.
    *   **Алдааг төвлөрүүлж барих**: Серверээс ирэх алдааны хариуг уншиж UI хэсэгт үзүүлэхэд хялбар болгон JavaScript `Error` болгож шиднэ.
2.  **Сонгосон шалтгаан**: Програмын хэмжээг хөнгөн байлгах, мөн Google Gemini AI-тай Server-Sent Events (SSE) ашиглан текст урсгах (streaming) харилцааг хялбар хийх.

---

## 6. Дундын Дансны Баталгаажуулалтын Логик (Escrow Flow)

1.  **Төлбөр түгжих**: Худалдан авагчийн балансаас мөнгө хасагдаж систем дэх дундын дансанд түр байрлана (`PENDING_SELLER_CREDS`).
2.  **Мэдээлэл оруулах**: Борлуулагч тоглоомын аккаунт болон шаардлагатай мэдээллээ системд оруулна (`PENDING_MIDDLEMAN_VERIFICATION`).
3.  **Админ шалгалт (Escrow Middleman)**: Админ эсвэл модератор мэдээллийг өөрийн биеэр орж шалгана:
    *   **Зөв байвал (Verify)**: Төлбөрийг борлуулагчийн баланс руу шилжүүлж, худалдан авагчид аккаунтын мэдээллийг илгээнэ (`COMPLETED`).
    *   **Буруу байвал (Cancel)**: Мөнгийг худалдан авагчид буцаан олгож гүйлгээг цуцална (`CANCELLED`).
